# Setup — hosting & schedule

The AI-Infrastructure Basket is a static page (`index.html` + `app.js` + `style.css`) that
renders from `data/series.json`. The data is regenerated by `update.py`.

## 1. Hosting (GitHub Pages)

This folder lives in the `claude-experiments` repo, which publishes to GitHub Pages from
`main`. The page is live at
`https://kstenson.github.io/claude-experiments/ai-infra-basket/`. Everything is static and
the JSON is fetched at runtime, so **every push that updates `data/series.json` redeploys
automatically** — no build step.

## 2. Schedule the routine

The engine is a **Claude Code on the web scheduled session**:

1. Open Claude Code on the web on this repository.
2. Create a daily recurring session (e.g. **22:30 UTC**, after the US market close).
3. Set the session prompt to the contents of `ai-infra-basket/.claude/daily-prompt.md`
   (or just: *"Run today's AI-Infra Basket update per ai-infra-basket/CLAUDE.md."*).
4. Ensure the environment's **network policy allows outbound HTTPS** (so `update.py` can
   reach Yahoo Finance, and web search works for Friday decisions) and that the session can
   **push** to `main`.

Daily runs refresh the prices; on a US trading Friday the session also makes and logs a
trade decision in `data/strategy.json`. Either way it runs `update.py`, validates, commits
and pushes — which redeploys the page.

> A single repo-wide "daily experiments" session can run this alongside the other
> experiments; it just needs to execute `python3 ai-infra-basket/update.py` and pick up the
> Friday step from the prompt.

## 3. Dependencies

`update.py` uses only the Python standard library (`urllib`, `json`) — no `pip install`
needed. `validate.mjs` needs Node (already used by the repo's validate workflow).

## 4. Local preview

```sh
python3 ai-infra-basket/update.py
cd ai-infra-basket && python3 -m http.server 8000
# open http://localhost:8000
```
